Create packages/contracts/src/governance/rbac.ts with:

1. PermissionSchema: object with
   - resource: string (e.g., "tasks", "runs", "governance/rules")
   - action: enum ["create", "read", "update", "delete", "approve", "reject", "execute", "admin"]
   - conditions: optional array of objects with field, operator, value for field-level checks

2. RoleSchema: object with
   - id: string.uuid()
   - name: string.min(1).max(100)
   - description: string.max(500).optional()
   - parentRoleId: string.uuid().optional() (for RBAC1 inheritance)
   - permissions: array of PermissionSchema
   - priority: number.int().min(0).max(1000).default(100)
   - isSystemRole: boolean.default(false) (for default roles)
   - createdAt: string.datetime()
   - updatedAt: string.datetime()

3. RoleHierarchySchema: object for resolving inheritance
   - roleId: string.uuid()
   - ancestorRoleIds: array of string.uuid() (all parent roles in chain)
   - depth: number.int().min(0) (inheritance depth)

4. SubjectSchema: object representing actor (user or system)
   - id: string
   - type: enum ["user", "service", "system"]
   - roles: array of string (role IDs assigned)
   - attributes: record of string->string for dynamic attributes (department, team, etc.)

5. PermissionCheckResultSchema: object with
   - granted: boolean
   - subjectId: string
   - resource: string
   - action: string
   - reason: string (human-readable explanation)
   - matchedPermissions: array of PermissionSchema (which permissions granted access)
   - deniedBy: optional object with ruleId, reason for explicit denials
   - checkedAt: string.datetime()

6. RoleAssignmentSchema: object for user-role mappings
   - id: string.uuid()
   - subjectId: string
   - subjectType: enum ["user", "service"]
   - roleId: string.uuid()
   - assignedBy: string (admin who assigned)
   - assignedAt: string.datetime()
   - expiresAt: string.datetime().optional() (for temporary access per SPEC-01)
   - isActive: boolean.default(true)

Export all Zod schemas and inferred types.

Update packages/contracts/src/governance/index.ts to export * from ./rbac.
</action>

5. extractSubjectFromRequest function:
   - Input: FastifyRequest
   - Returns: Subject | undefined
   - Extract from JWT token, session, or API key
   - Handle missing authentication gracefully

6. requirePermission middleware factory:
   - Input: permission: Permission, options?: { allowGuest?: boolean }
   - Returns: Fastify preHandler hook
   - Implementation:
     - Extract subject from request
     - If no subject and !allowGuest, return 401
     - Call checkPermission
     - If denied, return 403 with reason
     - If granted, attach result to request for downstream use

7. requireAnyPermission middleware:
   - Input: permissions: Permission[]
   - Returns: Fastify preHandler hook
   - Grant if ANY permission check passes

8. requireAllPermissions middleware:
   - Input: permissions: Permission[]
   - Returns: Fastify preHandler hook
   - Grant only if ALL permission checks pass

9. withPermissionCheck decorator:
   - Register Fastify decorator for permission checking in route handlers

   - Query params: includeSystem (boolean), search (string)

   - Create new role
   - Body: Omit<Role, "id" | "createdAt" | "updatedAt">
   - Validation: parentRoleId must exist if provided
   - Response: Created role
   - Permission: roles:create
   - Error 409 if role name exists

   PUT /api/roles/:id:
   - Update existing role
   - Params: id (UUID)
   - Body: Partial<Omit<Role, "id" | "createdAt" | "updatedAt">>

   - Response: Updated role
   - Permission: roles:update

   - Delete role
   - Params: id (UUID)
   - Validation: cannot delete system roles
   - Validation: cannot delete role with active assignments

   - Query params: subjectId, roleId, activeOnly (boolean)

   - Validation: check for duplicate active assignment
   - Response: Created assignment
   - Permission: roles:assign

   - Revoke role assignment
   - Params: id (UUID)
   - Response: 204 No Content
   - Permission: roles:revoke
