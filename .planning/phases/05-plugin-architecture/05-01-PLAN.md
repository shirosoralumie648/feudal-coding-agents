Define:

```typescript
export interface PluginDiscoveryFailure {
  pluginId?: string;
  manifestPath?: string;
  diagnostic: PluginDiagnostic;
}

export interface PluginDiscoveryResult {
  discovered: PluginRegistrationInput[];
  failed: PluginDiscoveryFailure[];
}

export class PluginDiscovery {
  constructor(options: { roots: string[] }) {}
  async discover(): Promise<PluginDiscoveryResult> {}
}
```

Discovery algorithm:

1. For each configured root, check whether `<root>/plugin.json` exists. If yes, treat `root` as one plugin directory.
2. If `<root>/plugin.json` does not exist, list direct children and include child directories with `<child>/plugin.json`.
3. If a configured root cannot be read, append failure `{ code: "PLUGIN_ROOT_NOT_FOUND", severity: "warning" }` and continue.
4. Read each `plugin.json` as UTF-8. If JSON.parse fails, append failure `{ code: "PLUGIN_MANIFEST_JSON_INVALID", severity: "error" }`.
5. Validate parsed JSON with `PluginManifestSchema.safeParse`. If invalid, append failure `{ code: "PLUGIN_MANIFEST_INVALID", severity: "error" }`.
6. Check duplicate ids within the same discovery pass. If duplicate, append failure `{ code: "PLUGIN_DUPLICATE_ID", severity: "error" }` for the duplicate manifest and do not include it in `discovered`.
7. Validate `manifest.entry.module`:
   - `path.isAbsolute(entry.module)` is unsafe.
   - `entry.module.split(/[\\\\/]/).includes("..")` is unsafe.
   - `access(path.join(pluginDir, entry.module))` must succeed.
8. For valid manifests, push:
   ```typescript
   {
     manifest,
     source: {
       kind: "local-directory",
       rootPath: pluginDir,
       manifestPath
     }
   }
   ```
