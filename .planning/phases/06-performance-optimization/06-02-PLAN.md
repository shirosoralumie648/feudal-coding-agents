<objective>
Reduce read-model and analytics audit fan-out while preserving event-sourced projection correctness.

Purpose: Close the Phase 6 N+1 requirement on the persisted read side. The critical fix is to give analytics and audit routes one bulk audit-event path instead of loading every task and then every task stream. The plan also removes unnecessary full task hydration from event/runs/artifact reads where only existence or child rows are required.

Output: `TaskStore` gains a bulk audit loading method, persisted and memory stores implement it, `AnalyticsService` prefers it, and tests prove missing-task semantics, replay behavior, and projection checkpoints remain intact.
</objective>

Implementation facts:
- `AnalyticsService.loadAuditEvents()` currently falls back to `tasks.map((task) => store.listTaskEvents(task.id))`.
- `task-read-model.listTaskEvents()` currently calls `this.getTask(taskId)` before `loadStream("task", taskId)`, which hydrates history, artifacts, and runs.
- `listTaskRuns()` and `listTaskArtifacts()` already query child tables, but they perform repeated inline existence checks.
- `replayTaskAtEventId()` depends on event ordering and must continue returning `undefined` for missing tasks or missing event ids.
</context>
