Implement Fastify routes in apps/control-plane/src/routes/analytics.ts:

**Export signature:**
```typescript
import type { FastifyInstance, FastifyRequest, FastifyReply } from "fastify";
import type { AnalyticsService } from "../services/analytics-service";
import type { AuditTrailQuery } from "@feudal/contracts";
import { AuditTrailQuerySchema } from "@feudal/contracts";
import { createPostgresEventStore } from "@feudal/persistence";

export function registerAnalyticsRoutes(
  app: FastifyInstance,
  options: {
    analyticsService: AnalyticsService;
    eventStore: ReturnType<typeof createPostgresEventStore>;
  }
)
```

**Inside:** Three route registrations:

1. **GET /analytics/snapshot** (async handler):
   - const snapshot = options.analyticsService.getLatestSnapshot()
   - If no snapshot: reply.status(503).send({ status: "no_data", message: "No metrics snapshot available yet" })
   - Otherwise: reply.send(snapshot)

2. **GET /analytics/stream** (async handler, manually manage reply):
   - reply.raw.writeHead(200, { "Content-Type": "text/event-stream", "Cache-Control": "no-cache", "Connection": "keep-alive" })
   - const unsubscribe = options.analyticsService.subscribe({ onMetricSnapshot(snapshot) { reply.raw.write(`data: ${JSON.stringify({type: "snapshot", payload: snapshot})}\n\n`) } })
   - Set up heartbeat interval: setInterval(() => { reply.raw.write(": heartbeat\n\n") }, 15000)
   - On request close (req.raw.on("close")): unsubscribe(), clearInterval(heartbeatId)

3. **GET /analytics/audit-trail** (async handler):
   - Parse query with AuditTrailQuerySchema; return 400 on failure
   - Load events: const events = await options.eventStore.loadAfter(query.cursor ?? 0)
   - Apply filters in sequence:
     a. taskId filter: events.filter(e => e.streamId === query.taskId)
     b. agentId filter: events.filter(e => e.metadataJson?.agentId === query.agentId)
     c. eventType filter: events.filter(e => e.eventType === query.eventType)
     d. timeRange filter: events.filter(e => { const t = new Date(e.occurredAt).getTime(); return t >= new Date(query.timeRange!.start).getTime() && t <= new Date(query.timeRange!.end).getTime() })
     e. searchQuery filter: maintain in-memory search index (Map<number, string> scoped to route closure). For each unfiltered event, index its payload as JSON.stringify(e.payloadJson).toLowerCase(). Filter: indexed value includes query.searchQuery.toLowerCase().
   - Slice to query.limit entries
   - Map to AuditTrailEntry: { eventId: e.id, streamType: e.streamType, streamId: e.streamId, eventType: e.eventType, occurredAt: e.occurredAt, payloadSummary: JSON.stringify(e.payloadJson).substring(0, 200), actorType: e.metadataJson?.actorType ?? null, actorId: e.metadataJson?.actorId ?? null }
   - Compute nextCursor: if filtered.length > 0, last entry's eventId; undefined if no more
   - Reply: { entries, nextCursor, totalCount: rawEvents.length }

**Edge cases:**
- SSE reconnection: client handles; server just writes new events when subscribed. No replay/backfill needed per D-06 (real-time only).
- Search index memory: cap at 50000 entries; when exceeded, clear oldest entries.
- Empty results: return { entries: [], totalCount: 0 } (no nextCursor).
