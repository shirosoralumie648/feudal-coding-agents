```
z.object({
  timestamp: z.string().datetime(),
  tasksByStatus: z.record(z.string(), z.number().int().nonnegative()),
  runsByAgent: z.record(z.string(), z.number().int().nonnegative()),
  runsByStatus: z.record(z.string(), z.number().int().nonnegative()),
  totalTaskCount: z.number().int().nonnegative(),
  totalRunCount: z.number().int().nonnegative(),
  awaitingApproval: z.number().int().nonnegative(),
  recoveryRequired: z.number().int().nonnegative(),
  avgApprovalLatencyMs: z.number().nonnegative().nullable(),
  errorRate: z.number().min(0).max(1),
  tokenUsage: SystemTokenUsageSummarySchema,
})
```
Infer type: `MetricSnapshot`.

3. **MetricListener interface** (TypeScript interface, NOT Zod schema):
```typescript
export interface MetricListener {
  onMetricSnapshot(snapshot: MetricSnapshot): void;
}
```

4. **MetricEventEmitter interface** (TypeScript interface, NOT Zod schema):
```typescript
export interface MetricEventEmitter {
  subscribe(listener: MetricListener): () => void;
  getLatestSnapshot(): MetricSnapshot | undefined;
}
```
Per D-16, AnalyticsService implements MetricEventEmitter, AlertService implements MetricListener.

5. **AlertRuleSchema:**
```
z.object({
  id: z.string(),
  name: z.string().min(1).max(200),
  description: z.string().max(1000).optional(),
  enabled: z.boolean().default(true),
  metricField: z.enum(["totalTaskCount", "awaitingApproval", "recoveryRequired", "errorRate", "avgApprovalLatencyMs"]),
  operator: z.enum(["gt", "gte", "lt", "lte", "eq"]),
  threshold: z.number(),
  suppressionWindowMs: z.number().int().positive().default(300000), // 5 min per D-15
  notificationChannels: z.array(z.enum(["in-app", "webhook"])).default(["in-app"]),
})
```
Infer type: `AlertRule`.

6. **AlertStateSchema:**
```
z.object({
  ruleId: z.string(),
  status: z.enum(["ok", "firing", "suppressed", "resolved"]),
  triggeredAt: z.string().datetime().optional(),
  resolvedAt: z.string().datetime().optional(),
  lastSuppressedAt: z.string().datetime().optional(),
  currentValue: z.number(),
  threshold: z.number(),
})
```
Infer type: `AlertState`.

7. **AlertEventSchema** (for in-app notifications per D-14):
```
z.object({
  id: z.string(),
  ruleId: z.string(),
  ruleName: z.string(),
  status: z.enum(["firing", "resolved"]),
  message: z.string(),
  metricValue: z.number(),
  threshold: z.number(),
  timestamp: z.string().datetime(),
})
```
Infer type: `AlertEvent`.

8. **WebhookPayloadSchema** (Slack/Discord-compatible per D-14):
```
z.object({
  text: z.string(),
  attachments: z.array(z.object({
    color: z.enum(["danger", "good", "warning"]),
    title: z.string(),
    fields: z.array(z.object({
      title: z.string(),
      value: z.string(),
      short: z.boolean().default(true),
    })),
    footer: z.string().optional(),
  })).optional(),
})
```

9. **AuditTrailQuerySchema:**
```
z.object({
  taskId: z.string().optional(),

    end: z.string().datetime(),

  searchQuery: z.string().max(500).optional(),
  limit: z.number().int().min(1).max(500).default(100),

Infer type: `AuditTrailQuery`.

])
```

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Alert rule JSON file | User-supplied configuration loaded at runtime; thresholds and metric fields must be Zod-validated |
| Webhook URL | User-supplied URL in configuration; must be validated as a proper URL format |
| Audit trail search query | Free-text input from web UI; must be max 500 chars per AuditTrailQuerySchema |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Spoofing | AlertRuleSchema | mitigate | Zod parsing at config load time validates all rule fields; unknown fields rejected by Zod strip |
| T-04-02 | Tampering | MetricSnapshotSchema | mitigate | Snapshots are generated server-side from authenticated data; no external input accepted |
| T-04-03 | Injection | AuditTrailQuerySchema | mitigate | searchQuery limited to 500 chars; all query fields are parameterized in SQL; Zod validates shape before use |
| T-04-04 | Denial of Service | AuditTrailQuerySchema | mitigate | limit clamped to max 500 per query; cursor-based pagination prevents full table scans |
</threat_model>
