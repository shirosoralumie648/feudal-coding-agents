    Implementation rules for each method:

    **createTemplate(input):**
    1. If `templates.has(input.name)`, throw `new Error(\`Template "\${input.name}" already exists\`)`
    2. Create template with status "draft", eventVersion 1, createdAt/updatedAt = now ISO
    3. Store in templates Map, record `template.created` version event
    4. Return the created template

    **getTemplate(name):**
    1. Return `templates.get(name)` or undefined

    **updateTemplate(name, updates, expectedVersion):**
    D-13: optimistic locking
    1. Get existing template -- if not found, throw `new Error(\`Template "\${name}" not found\`)`
    2. If `existing.eventVersion !== expectedVersion`, throw `new Error(\`Version mismatch for template "\${name}": expected v\${expectedVersion}, current v\${existing.eventVersion}\`)`
    3. Merge updates (only update the fields provided in `updates`: version, parameters, steps -- never mutate name, createdAt, eventVersion)
    4. Increment eventVersion, set updatedAt = now ISO
    5. If status is "published" and the template is in published cache, update the cache entry
    6. Record `template.updated` version event
    7. Return updated template

    **publishTemplate(name, expectedVersion):**
    D-13: draft -> published transition
    1. Get template, validate expectedVersion (same as updateTemplate step 2)
    2. If `template.status === "published"`, throw `new Error(\`Template "\${name}" is already published\`)`
    3. Set status to "published", set `lastPublishedVersion = expectedVersion + 1`, increment eventVersion, update updatedAt
    4. Add to publishedCache (D-14 memory cache for fast reads)
    5. Record `template.published` version event
    6. Return updated template

    **unpublishTemplate(name, expectedVersion):**
    1. Get template, validate expectedVersion
    2. If `template.status !== "published"`, throw `new Error(\`Template "\${name}" is not published\`)`
    3. Set status to "draft", increment eventVersion, update updatedAt
    4. Remove from publishedCache
    5. Record `template.unpublished` version event
    6. Return updated template

    **deleteTemplate(name, expectedVersion):**
    1. Get template, validate expectedVersion
    2. If `template.status === "published"`, throw `new Error(\`Cannot delete published template "\${name}". Unpublish it first.\`)`
    3. Remove from templates Map, remove from publishedCache
    4. Record `template.deleted` version event (still stored in versionHistory Map for audit trail)
    5. Return void

    **listTemplates(filter?):**
    1. Get all templates from Map values
    2. If filter.status is set, filter by that status
    3. Return filtered array

    **getTemplateVersionHistory(name):**
    1. Return `versionHistory.get(name) ?? []`

    Follow the project conventions: explicit return types on public methods, `readonly` on interface fields, `unknown` over `any`.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| TemplateImport boundary | External JSON files (TemplateExportPackage) cross into the store -- format version checked, name collision rejected, status reset to draft on import |
| TemplateExport boundary | Internal template data crosses to external files -- status/eventVersion/lastPublishedVersion are stripped; only published templates are exportable |
| Concurrent update boundary | Multiple callers may attempt to modify the same template -- optimistic locking via expectedVersion prevents lost updates |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-10 | S (Spoofing) | TemplateImport -- package format | mitigate | importTemplate validates that `pkg.format === "feudal-template/v1"` before processing; unknown formats are rejected with a descriptive error |
| T-03-11 | T (Tampering) | TemplateImport -- status injection | mitigate | importTemplate resets status to "draft" regardless of what the export package contains; the importer cannot inject published status |
| T-03-12 | T (Tampering) | TemplateExport -- internal state leakage | mitigate | exportTemplate strips status, eventVersion, and lastPublishedVersion from the exported template; internal state cannot leak to shared files |
| T-03-13 | I (Information Disclosure) | TemplateExport -- draft access | mitigate | exportTemplate rejects draft templates; only published templates can be exported, preventing premature/internal template leakage |
| T-03-14 | T (Tampering) | updateTemplate -- concurrent writes | mitigate | Optimistic locking via expectedVersion check: if two callers read version N and both try to write, the second caller's write is rejected with a version mismatch error |
</threat_model>
